Авторизация

Процесс "узнавания" зарегистрированного пользователя в какой-либо системе, обычно для этого ему нужно указать свое учетное имя и пароль. Пользователь, успешно прошедший авторизацию (введший правильное учетное имя и пароль) называется авторизованным.

Система управления сайтом (CMS)

Программный продукт позволяющий пользователям, не обладающим навыками разработки веб-сайтов, управлять содержимым, структурой и внешним видом веб-сайта с помощью интуитивно понятного интерфейса.

META-теги

Специальные скрытые элементы HTML-страницы, позволяющие поисковым системам получать общую информацию о ее содержимом. Правильно составленные META-теги страницы позволяют посетителям интернета легче находить ее с помощью поисковых систем.

Файл robots.txt

Специальный файл для поисковых систем индексирующих сайт, в котором им даются рекомендации о том, какие разделы сайта не следует просматривать и индексировать. Это дает возможность скрывать разделы сайта от пользователей интернета.

Портал

Совокупность веб-сайтов объединенных в общее веб-пространство, имеющих схожий интерфейс и функционально связанных друг с другом. Как правило для регистрации на разных сайтах портала используется общая база клиентов, что позволяет зарегистрировавшись на одном из них получить доступ и ко всем остальным.

Спам

Назойливая корреспонденция электронной почты рекламного характера, одна из самых насущных и пока не решенных проблем сети интернет. Для борьбы со спамом используются спам-фильтры, которые позволяют отсеивать большую его часть, но требуют контроля человеком, т.к. срабатывают не всегда правильно. Самый простой и в то же время эффективный способ уменьшения потока спама - не публиковать свой адрес электронной почты в открытых источниках в сети.

Титул страницы

HTML-тег содержащий название страницы, которое отображается в заголовке окна браузера. Это название имеет большое значение для поисковых систем.

URL

Универсальный указатель ресурса.

Представляет собой адрес страницы, файла или другого ресурса в сети, подобные адреса отображаются в адресной строке браузера. Адрес выглядит примерно так: http://www.site.ru/news/2006/05/11/newsname.html и состоит из трех частей:

http

протокол соединения с сервером

www.site.ru

имя сайта

/news/2006/05/11/newsname.html

путь к файлу на сервере - несколько вложенных папок и имя файла
